# 项目对比文档

本仓库包含两个音乐播放器项目，分别针对不同平台开发。

## 项目概览

### 1. Android音乐播放器
**目录**: `app/`
**平台**: Android (移动端)
**语言**: Kotlin
**最低版本**: Android 7.0 (API 24)

### 2. macOS音乐播放器
**目录**: `mac-music-player/`
**平台**: macOS (桌面端)
**语言**: Python
**最低版本**: macOS 10.14+, Python 3.8+

## 功能对比

| 功能 | Android版 | macOS版 |
|------|-----------|---------|
| 本地音乐导入 | ✅ | ✅ |
| 多种排序方式 | ✅ (8种) | ✅ (8种) |
| 播放模式 | ✅ (3种) | ✅ (3种) |
| 歌词显示 | ✅ | ✅ |

| 点击歌词跳转 | ✅ | ✅ |

| 专辑封面显示 | ✅ | ✅ |
| 后台播放 | ✅ | ✅ |
| 通知控制 | ✅ | ❌ |
| 音频焦点管理 | ✅ | ❌ |

## 技术栈对比

### Android版
```
架构: MVVM
语言: Kotlin
UI框架: Material Design 3
数据库: Room (SQLite)
播放引擎: ExoPlayer
依赖注入: Hilt
异步处理: Coroutines
```

### macOS版
```
架构: MVC
语言: Python
UI框架: PyQt6
数据库: SQLite (原生)
播放引擎: pygame.mixer
元数据提取: mutagen
```

## 核心逻辑对比

### 1. 数据模型

**Android (Kotlin)**
```kotlin
@Entity(tableName = "songs")
data class Song(
    @PrimaryKey val id: Long,
    val title: String,
    val artist: String,
    // ...
)
```

**macOS (Python)**
```python
@dataclass
class Song:
    id: int
    title: str
    artist: str
    # ...
```

### 2. 播放器核心

**Android**
- 使用ExoPlayer，专业级音频播放
- 支持多种音频格式和编解码器
- 内置缓冲和流媒体支持

**macOS**
- 使用pygame.mixer，轻量级音频播放
- 支持常见音频格式
- 简单直接的API

### 3. 歌词解析

两个版本使用相同的逻辑：
- 支持LRC格式（带时间戳）
- 支持纯文本格式
- 正则表达式解析时间戳
- 实时同步显示

### 4. 数据持久化

**Android**
- Room数据库（ORM）
- 类型安全的查询
- 自动生成SQL

**macOS**
- 原生SQLite
- 手动SQL查询
- 更灵活的控制

## 选择建议

### 选择Android版，如果你：
- 需要移动端音乐播放器
- 要求专业级音频质量
- 需要后台播放和通知控制
- 计划发布到应用商店

- 需要桌面端音乐播放器
- 想要快速开发和部署

- 计划跨平台支持

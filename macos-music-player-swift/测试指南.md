# Swift macOS 音乐播放器 - 测试指南

## 快速测试步骤

### 1️⃣ 编译应用

```bash
cd macos-music-player-swift
xcodebuild -project MusicPlayer.xcodeproj -scheme MusicPlayer -configuration Release clean build
```

预期输出：
```
** BUILD SUCCEEDED **
```

### 2️⃣ 启动应用

```bash
open ~/Library/Developer/Xcode/DerivedData/MusicPlayer-*/Build/Products/Release/MusicPlayer.app
```

或者在Xcode中：
```bash
open MusicPlayer.xcodeproj
# 按 Cmd+R 运行
```

### 3️⃣ 测试专辑封面尺寸

**测试目标：** 验证播放控制按钮不再被遮挡

**步骤：**
1. 点击"添加音乐文件夹"按钮
2. 选择包含音乐文件的文件夹
3. 等待扫描完成
4. 双击任意歌曲开始播放
5. 观察播放器界面

**预期结果：**
- ✅ 专辑封面尺寸为 120x120 像素
- ✅ 播放/暂停按钮（大圆形按钮）完全可见
- ✅ 上一首/下一首按钮完全可见
- ✅ 所有控制按钮都可以正常点击
- ✅ 界面布局平衡美观

**对比：**
- ❌ 之前：150x150，按钮被遮挡
- ✅ 现在：120x120，按钮完全可见

### 4️⃣ 测试歌词搜索功能

**测试目标：** 验证在线歌词搜索功能正常工作

**步骤：**
1. 播放一首没有歌词的歌曲
2. 点击"搜索歌词"按钮（放大镜图标）
3. 在Xcode Console中观察日志输出
4. 等待几秒钟
5. 检查歌词面板

**预期日志输出：**
```
🔍 搜索歌词: [歌曲名] [歌手名]
📡 请求URL: https://music-api.heheda.top/search?keywords=...
📥 响应状态码: 200
📄 响应内容: {"result":{"songs":[...
✅ 找到歌曲ID: 123456
📡 请求歌词URL: https://music-api.heheda.top/lyric?id=123456
📥 歌词响应状态码: 200
✅ 歌词获取成功，长度: 1234 字符
```

**预期结果：**
- ✅ 控制台显示详细的搜索日志
- ✅ 歌词成功下载
- ✅ 歌词显示在右侧面板
- ✅ 歌词随播放进度同步滚动
- ✅ 当前歌词行高亮显示

**如果失败：**
- ❌ 检查网络连接
- ❌ 查看Console中的错误信息
- ❌ 确认API地址是否正确

### 5️⃣ 测试歌词保存功能

**测试目标：** 验证歌词能够保存到本地

**步骤：**
1. 搜索并获取歌词（参考步骤4）
2. 完全退出应用（Cmd+Q）
3. 重新启动应用
4. 播放同一首歌曲
5. 观察歌词是否自动加载

**预期结果：**
- ✅ 歌词自动显示，无需再次搜索
- ✅ 歌词内容与之前一致
- ✅ 歌词同步正常

**歌词保存位置：**
```
~/Library/Containers/com.ijackey.MusicPlayer/Data/Library/Application Support/MusicPlayer/lyrics/
```

### 6️⃣ 测试多种场景

#### 场景A：有封面的歌曲

- 检查封面是否正确显示
- 检查尺寸是否为120x120

#### 场景D：英文歌曲

- 检查英文显示是否正常
- 检查歌词同步是否准确

### 7️⃣ 性能测试

**测试目标：** 验证应用性能

**指标：**
- 启动时间：< 1秒
- 内存占用：< 100MB
- CPU占用：播放时 < 10%
- 歌词搜索响应：< 5秒

**测试方法：**
1. 使用Activity Monitor监控资源使用
2. 测量启动到界面显示的时间
3. 测量歌词搜索的响应时间

## 常见问题排查

### 问题1：编译失败

**症状：**
```
** BUILD FAILED **
```

**解决方案：**
1. 检查Xcode版本（需要15.0+）
2. 检查macOS版本（需要12.0+）
3. 清理构建缓存：
   ```bash
   rm -rf ~/Library/Developer/Xcode/DerivedData/MusicPlayer-*
   ```
4. 重新编译

### 问题2：应用无法启动

**症状：**
- 双击应用无反应
- 或显示"应用已损坏"

**解决方案：**
1. 检查代码签名：
   ```bash
   codesign -vv ~/Library/Developer/Xcode/DerivedData/MusicPlayer-*/Build/Products/Release/MusicPlayer.app
   ```
2. 允许运行未签名应用：
   ```bash
   xattr -cr ~/Library/Developer/Xcode/DerivedData/MusicPlayer-*/Build/Products/Release/MusicPlayer.app
   ```

### 问题3：歌词搜索失败

**症状：**
- 点击"搜索歌词"无反应
- 或显示"未找到歌词"

**排查步骤：**
1. 检查网络连接：
   ```bash
   curl -I https://music-api.heheda.top/
   ```
2. 查看Console日志：
   - 打开Xcode
   - Window > Devices and Simulators
   - 选择Mac
   - 查看Console输出
3. 检查API响应：
   ```bash
   curl "https://music-api.heheda.top/search?keywords=告白气球&type=1&limit=20"
   ```

### 问题4：专辑封面仍然遮挡按钮

**症状：**
- 播放控制按钮被遮挡
- 无法点击播放/暂停按钮

**排查步骤：**
1. 确认代码已更新：
   ```bash
   grep "width: 120" macos-music-player-swift/MusicPlayer/Views/PlayerView.swift
   ```
   应该看到：`.frame(width: 120, height: 120)`
2. 确认已重新编译
3. 确认运行的是新版本

### 问题5：歌词不同步

**症状：**
- 歌词显示但不滚动
- 歌词滚动但不准确

**排查步骤：**
1. 检查LRC文件格式
2. 检查时间戳格式：`[mm:ss.xx]`
3. 查看Console中的解析日志

## 自动化测试（可选）

如果需要自动化测试，可以使用以下脚本：

```bash
#!/bin/bash

echo "🧪 开始自动化测试..."

# 1. 编译测试
echo "1️⃣ 编译测试..."
xcodebuild -project MusicPlayer.xcodeproj -scheme MusicPlayer -configuration Release clean build > /dev/null 2>&1
if [ $? -eq 0 ]; then
    echo "✅ 编译成功"
else
    echo "❌ 编译失败"
    exit 1
fi

# 2. 代码检查
echo "2️⃣ 代码检查..."
if grep -q "width: 120" MusicPlayer/Views/PlayerView.swift; then
    echo "✅ 专辑封面尺寸正确"
else
    echo "❌ 专辑封面尺寸错误"
fi

if grep -q "music-api.heheda.top" MusicPlayer/Services/LyricsAPI.swift; then
    echo "✅ API地址正确"
else
    echo "❌ API地址错误"
fi

# 3. 网络测试
echo "3️⃣ 网络测试..."
if curl -s -I https://music-api.heheda.top/ | grep -q "200\|301\|302"; then
    echo "✅ API服务可访问"
else
    echo "⚠️  API服务不可访问"
fi

echo "🎉 自动化测试完成！"
```

保存为 `test.sh`，然后运行：
```bash
chmod +x test.sh
./test.sh
```

# Swift macOS 音乐播放器 - 修复说明

## 修复时间：2026-03-05

## 修复内容

### 1. ✅ 缩小专辑封面尺寸

**问题：** 播放控制按钮被专辑封面遮挡

**解决方案：**
- 将专辑封面尺寸从 150x150 缩小到 120x120
- 同时调整了应用图标显示尺寸（降级方案）
- 调整了音乐图标字体大小（从60改为50）

**修改文件：**
- `MusicPlayer/Views/PlayerView.swift`

**修改位置：**
```swift
// 之前：
.frame(width: 150, height: 150)

// 之后：
.frame(width: 120, height: 120)
```

### 2. ✅ 修复歌词搜索功能

**问题：** 搜索歌词没有反应

**根本原因：**
- Swift版本使用的API地址不正确
- 之前使用：`https://music.163.com/api/search/get/web`
- Android版本使用：`https://music-api.heheda.top/search`

**解决方案：**
- 更新API基础URL为 `https://music-api.heheda.top`
- 修改搜索接口为 `/search`
- 修改歌词接口为 `/lyric`
- 调整请求参数名称（`s` → `keywords`）
- 移除不必要的参数（`lv`, `tv`, `offset`）
- 增加详细的日志输出（使用emoji标记）
- 增加超时时间设置（30秒）

**修改文件：**
- `MusicPlayer/Services/LyricsAPI.swift`

**API对比：**

| 项目 | 旧版本（错误） | 新版本（正确） |
|------|--------------|--------------|
| 基础URL | `music.163.com` | `music-api.heheda.top` |
| 搜索接口 | `/api/search/get/web` | `/search` |
| 歌词接口 | `/api/song/lyric` | `/lyric` |
| 搜索参数 | `s`, `offset` | `keywords` |
| 歌词参数 | `id`, `lv`, `tv` | `id` |

**新增日志输出：**
```
🔍 搜索歌词: [歌曲名] [歌手名]
📡 请求URL: [完整URL]
📥 响应状态码: [HTTP状态码]
📄 响应内容: [JSON内容预览]
✅ 找到歌曲ID: [歌曲ID]
✅ 歌词获取成功，长度: [字符数] 字符
❌ [错误信息]
```

## 与Android版本的一致性

### API配置对比

**Android版本 (NetworkModule.kt):**
```kotlin
@Provides
@Singleton
@LyricsRetrofit
fun provideLyricsRetrofit(okHttpClient: OkHttpClient): Retrofit {
    return Retrofit.Builder()
        .baseUrl("https://music-api.heheda.top/")
        .client(okHttpClient)
        .addConverterFactory(GsonConverterFactory.create())
        .build()
}
```

**Swift版本 (LyricsAPI.swift):**
```swift
private static let baseURL = "https://music-api.heheda.top"
private static let searchAPI = "\(baseURL)/search"
private static let lyricsAPI = "\(baseURL)/lyric"
```

### API接口对比

**Android版本 (LyricsApi.kt):**
```kotlin
interface MusicSearchApi {
    @GET("search")
    suspend fun searchMusic(
        @Query("keywords") keywords: String,
        @Query("type") type: Int = 1,
        @Query("limit") limit: Int = 20
    ): Response<MusicSearchResponse>
}

interface LyricsApi {
    @GET("lyric")
    suspend fun getLyrics(
        @Query("id") id: Long
    ): Response<LyricsResponse>
}
```

**Swift版本 (LyricsAPI.swift):**
```swift
// 搜索接口
components.queryItems = [
    URLQueryItem(name: "keywords", value: keywords),
    URLQueryItem(name: "type", value: "1"),
    URLQueryItem(name: "limit", value: "20")
]

// 歌词接口
components.queryItems = [
    URLQueryItem(name: "id", value: "\(songId)")
]
```

✅ 现在Swift版本与Android版本完全一致！
